Linux Directory Structure

The linux system directory is really different from Windows. If you are using linux first time, you need to take some time to familiar with the file system structure.

  • / – The Root Directory, everything on Linux system is located under the / directory, known as the root directory.
  • /home – Home Folders, contains a home folder for each user, and it is also the FTP site directory.
  • /bin – Essential User Binaries, contains the essential user binaries (programs) that must be present when the system is mounted in single-user mode. It contains the binaries (programs) of user.
  • /sbin – System Administration Binaries, is similar to the /bin directory. It contains essential binaries that are generally intended to be run by the root user for system administration.
  • /mnt – Temporary Mount Points, is where system administrators mounted temporary file systems while using them. It is similar to CD drive and Floppy Disk.
  • /boot – Static Boot Files, contains the files needed to boot the system – for example, the GRUB boot loader’s files and Linux kernels are stored here.
  • /etc – Configuration Files, contains configuration files, which can generally be edited by hand in a vim editor. Note that the /etc/ directory contains system-wide configuration files – user-specific configuration files are located in each user’s home directory.
  • /var – Variable Data Files, contains inconstant files. It is the writable counterpart to the /usr directory, which must be read-only in normal operation.
  • /usr – User Binaries & Read-Only Data, the /usr directory contains applications and files used by users, as opposed to applications and files used by the system. The /usr/local directory is where locally compiled applications install to by default – this prevents them from mucking up the rest of the system.

pwd - Shows what directory (folder) you are in.
cd - Changes directories.
ls - Lists files.
ls -al - You’ll see a huge list of files that you can’t see with the ‘ls’ command alone and lots of details.
mkdir - Creates a directory.
rmdir - Removes a directory.
touch - Creates new, empty files.
cp - Copies file.
cp -r dir1 dir2 - Copies file recursively.

date - Prints out, or change the value of, the system’s time and date information. ex: date [OPTION]… [+FORMAT] ; date [-u|–utc|–universal] [MMDDhhmm[[CC]YY][.ss]]

Linux Users and Groups

Linux operating systems have the ability to multitask in a manner similar to other operating systems. However, Linux’s major difference from other operating systems is its ability to have multiple users. Linux was designed to allow more than one user to have access to the system at the same time. In order for this multiuser design to work properly, there needs to be a method to protect users from each other. This is where permissions come in to play.

Working with Users, Groups, and Directories

Creating and Deleting User Accounts

useradd Tom - To create a new standard user, use the useradd command.
passwd Tom - To set a password for the new user by using the passwd command. Note, you will need root privileges to change a user password.
userdel Tom - To remove a user account by using the userdel command.

Working with Groups

Each user in the Linux must belong to a group, not independent of the group. And the user and group are created by the root user. Linux uses groups as a way to organize users. Groups organize collections of accounts, primarily as a security measure. Control of group membership is administered through the /etc/group file, which shows a list of groups and its members. Every user has a default or primary group. When a user logs in, the group membership is set for their primary group. This means that when a user launches a program or creates a file, both the file and the running program will be associated with the user’s current group membership. A user may access other files in other groups, as long as they are also a member of that group and the access permissions are set. To run programs or create a file in a different group, the user must run the newgrp command to switch their current group.

groupadd - To add a new group.
cat /etc/group/ - To show a list of groups and its members.
useradd -g [groupName] [userName] - To create a new standard user, and distribute this user to the specific group.
usermod -g [groupName] [userName] - To change the user to a new group.
cat /etc/passwd - To show the information of users.

Directory and File Permissions

When a user creates a new file, the group of the file is the group of the user.
ls -ahl - To view file permissions and ownership on files and directories.
chgrp [groupName] [fileName] - To change the group of the file.
chown [userName] [fileName] - To change the file ownership.
To view file permissions and ownership on files and directories, use the ls -al command. The a option is to show hidden files or all files, and the l option is for the long listing. The output will be similar to the following:

The first column with the ten letters and dashes shows the permissions of the file or directory. The second column (with the single number) indicates the number of files or directories contained in the directory. The next column indicates the owner, followed by the group name, the size, date, and time of last access, and finally the name of the file . For example, using the first line from the output above, the details are as follows:

-rwxrw-r– are the permissions. First character - denotes whether it is a directory or a file, directory(d), link(l). The other nine characters rwxrw-r– are the permissions for the file/directory. The first three characters are for the user, the next three are for the group, and the last three are for others, r - read, w - write, x - execute, and dash - denotes permissions are removed. The example rwxrw-r– is broken down as follows:
The first three characters rwx - the user has read, write, and execute permissions. The next three characters rw- - the group has read and write permissions. The last three characters r– - all others have read only permissions.
1 is the number of files or directories.
root is the owner.
root is the group.
1213 is the size (byte).
Feb 2 09:39 is the date/time of last access.
abc is the directory.

Note that to use the octal format, we have to calculate the permissions for each portion of the file or directory. The first ten characters mentioned above will correspond to a four digit numbers in octal. The execute permission is equal to the number one (1), the write permission is equal to the number two (2), and the read permission is equal to the number four (4). Therefore, when you use the octal format, you will need to calculate a number between 0 and 7 for each portion of the permission. A table has been provided below for clarification.

Octal Value    Read    Write    Execute 
7              r       w        x       
6              r       w        -       
5              r       -        x       
4              r       -        -       
3              -       w        x       
2              -       w        -       
1              -       -        x       
0              -       -        -
