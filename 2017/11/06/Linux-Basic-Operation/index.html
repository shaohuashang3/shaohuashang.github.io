Linux Directory Structure

The linux system directory is really different from Windows. If you are using linux first time, you need to take some time to familiar with the file system structure.

  • / – The Root Directory, everything on Linux system is located under the / directory, known as the root directory.
  • /home – Home Folders, contains a home folder for each user, and it is also the FTP site directory.
  • /bin – Essential User Binaries, contains the essential user binaries (programs) that must be present when the system is mounted in single-user mode. It contains the binaries (programs) of user.
  • /sbin – System Administration Binaries, is similar to the /bin directory. It contains essential binaries that are generally intended to be run by the root user for system administration.
  • /mnt – Temporary Mount Points, is where system administrators mounted temporary file systems while using them. It is similar to CD drive and Floppy Disk.
  • /boot – Static Boot Files, contains the files needed to boot the system – for example, the GRUB boot loader’s files and Linux kernels are stored here.
  • /etc – Configuration Files, contains configuration files, which can generally be edited by hand in a vim editor. Note that the /etc/ directory contains system-wide configuration files – user-specific configuration files are located in each user’s home directory.
  • /var – Variable Data Files, contains inconstant files. It is the writable counterpart to the /usr directory, which must be read-only in normal operation.
  • /usr – User Binaries & Read-Only Data, the /usr directory contains applications and files used by users, as opposed to applications and files used by the system. The /usr/local directory is where locally compiled applications install to by default – this prevents them from mucking up the rest of the system.

Basic Linux Commands

Linux Most Frequently Used Commands
pwd - Shows what directory (folder) you are in.
cd - Changes directories.
ls - Lists files.
ls -al - You’ll see a huge list of files that you can’t see with the ‘ls’ command alone and lots of details.
mkdir - Creates a directory.
rmdir - Removes a directory.
touch - Creates new, empty files.
cp - Copies file.
cp -r dir1 dir2 - Copies file recursively.
mv data1 newdata/ - Moves the file data1 to the folder newdata and deletes the old one.
rm- Deletes the file in the current directory.
rm -rf - Force to deletes all content, including files and directories.
ln - Creates links between files.
ln -s - Creates symbolic links with the -s option
more - Displays output in the terminal one page at a time.
grep - Matches a regular expression against text in a file. ex: grep [OPTIONS] PATTERN [FILE…]
> - Redirect command. ex: cat test test1 test2 > test3 (This will create a file called test3 and all output will be redirected in a newly created file.)
>> - Appends standard output with redirection operator.
| - Pipe command. ex: cat song.txt | more (use parameters more with cat command as show above.)
find - Lists out all the files in the current directory as well as the subdirectories in the current directory. ex: find / -name aaa.c
date - Prints out, or change the value of, the system’s time and date information. ex: date [OPTION]… [+FORMAT] ; date [-u|–utc|–universal] [MMDDhhmm[[CC]YY][.ss]]
cal 11 2017 - Displays the calender for November of the year 2017.
cal 2018 - Displays the calender for whole year 2018.

Network State Commands
netstat -an - Shows information about all active connections to the server, including the source and destination IP addresses and ports, if you have proper permissions.
netstat -anp - And shows the PID (process identifier) and name of the program to which each socket belongs.
ping - It is frequently used to test, at the most basic level, whether another system is reachable over a network, and if so, how much time it takes for that data to be exchanged.
traceroute - Prints the route that packets take to a network host.
route - Shows or manipulates the IP routing table.
ifconfig - Views the configuration of, a network interface.
